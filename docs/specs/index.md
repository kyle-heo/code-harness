# 제품 스펙 인덱스

> 모든 기능 스펙은 이 파일을 통해 탐색하십시오.
> 스펙이 존재하지 않는 기능은 구현하지 마십시오. 스펙 먼저 작성 후 구현.

---

## 스펙 작성 원칙

- 스펙은 **"무엇을"** 기술, **"어떻게"** 는 기술하지 않음
- 수용 기준(Acceptance Criteria)을 검증 가능한 형태로 명시
- 에이전트가 스펙만 읽고 구현 판단이 가능해야 함
- 스펙 변경 시 반드시 관련 exec-plan도 갱신

## 스펙 파일 템플릿

새 스펙 작성 시 아래 형식을 사용하십시오:

```markdown
# [기능명] 스펙

## 개요
[1-2문장 기능 설명]

## 사용자 스토리
As a [사용자 유형], I want to [행동], so that [목적].

## 수용 기준 (Acceptance Criteria)
- [ ] AC-001: [검증 가능한 조건]
- [ ] AC-002: [검증 가능한 조건]

## 범위 외 (Out of Scope)
- [명시적으로 포함하지 않는 것]

## 관련 문서
- 설계 문서: `../design-docs/`
- 실행 플랜: `../plans/exec-plans/active/`
```
