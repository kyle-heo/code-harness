# QUALITY_SCORE.md
> 코드 품질 평가 기준입니다. `gc-jobs/quality-grader.md`가 주기적으로 점수를 산정합니다.

---

## 평가 항목 및 가중치

| 항목 | 가중치 | 측정 방법 |
|------|--------|-----------|
| 테스트 커버리지 | 30% | 브랜치 커버리지 (목표: 80% 이상) |
| 린터 위반 없음 | 25% | CONSTRAINTS.md 전 항목 통과 |
| 레이어 의존성 준수 | 20% | `linters/layer-deps.js` 결과 |
| 공유 유틸리티 활용 | 15% | 중복 구현 감지 건수 역산 |
| 문서화 최신성 | 10% | `gc-jobs/doc-staleness-check.md` 결과 |

## 점수 계산

```
품질 점수 = Σ(항목 점수 × 가중치)
항목 점수 = 100 - (위반 건수 × 패널티)
```

## 리팩터링 트리거 기준

| 점수 구간 | 조치 |
|-----------|------|
| 90-100 | 유지 |
| 80-89 | 다음 스프린트 내 개선 |
| 70-79 | `auto-refactor-pr.md` 즉시 트리거 |
| 70 미만 | 사람 에스컬레이션 + 즉시 트리거 |

## 황금 원칙 (Golden Rules)

1. 공유 유틸리티 패키지(`packages/shared/`) 우선 사용
2. 직접 만든 헬퍼는 불변 조건 중앙 관리 불가 -> 금지
3. 타입 안전성: `unknown` 타입 사용 후 타입 가드 적용
4. 테스트는 구현과 함께 같은 PR에서 작성
